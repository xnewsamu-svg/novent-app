Auditoría de MVP — CTO Assessment
¿Está Novent listo para producción?
Fecha: Junio 2026
SaaS Multiempresa para pequeños negocios

Resumen Ejecutivo

Novent es un SaaS multiempresa con Next.js 16 + Firebase. Tiene UI atractiva, estructura de datos sólida (companies/{id}/clientes, ventas, inventario) y los módulos core esbozados. Pero NO es un MVP comercializable. Hay bugs críticos que rompen auth, navegación y registro. La seguridad es inexistente (sin Firestore Rules). Funcionalidades clave incompletas.

40%
Puntaje de readiness
Con ~3 semanas de trabajo enfocado sería piloteable con 1-2 negocios reales.

1. Bugs Críticos Prohiben producción

Estos bugs IMPIDEN el uso en producción. Deben resolverse antes de cualquier otro trabajo.

C1 · Cookie name mismatch

Middleware auth nunca funciona

saveSession() escribe la cookie como 'firebase-auth-token' pero el middleware lee 'token'. El guard de rutas protegidas NUNCA funciona — cualquiera accede a /dashboard sin autenticación.

📁 lib/authToken.ts:11 vs middleware.ts:23⏱ 15 min
C2 · Archivo mal nombrado

pge.tsx en vez de page.tsx

Next.js requiere page.tsx para rutas dinámicas. El detalle de cliente en /clientes/[id] produce 404.

📁 app/clientes/[id]/pge.tsx⏱ 5 min
C3 · Link roto

Login → Signup no funciona

El login navega a /signup pero la carpeta se llama singup (sin la 'i'). La ruta real es /singup. Error 404 al intentar registrarse.

📁 app/login/page.tsx:116 → /signup vs app/singup/⏱ 5 min
C4 · Seguridad

Sin Firestore Security Rules

Cualquier usuario autenticado puede leer y escribir TODOS los datos de TODAS las empresas. No existe aislamiento multiempresa. Es el riesgo más grave.

📁 Proyecto completo⏱ 4 h
C5 · Exposición de datos

Password sin type="password" en signup

El input de contraseña en el formulario de registro muestra la contraseña en texto plano al escribir.

📁 app/singup/page.tsx:128-132⏱ 2 min
C6 · Configuración

Firebase config hardcodeada

API key, authDomain, projectId están hardcodeados en el bundle cliente. Sin .env ni variables de entorno.

📁 lib/firebase.ts:5-12⏱ 30 min

Conclusiones

¿Novent ya es un MVP?

No. No es un MVP usable en producción.

Tiene la apariencia de un MVP (UI bonita, landing page, estructura multiempresa), pero tiene bugs críticos que lo rompen completamente:

  • ❌ El middleware auth no funciona — cualquiera accede a rutas protegidas
  • ❌ El detalle de cliente no carga — pge.tsx no es page.tsx
  • ❌ El registro no funciona — link /signup vs ruta /singup
  • ❌ No hay security rules — base de datos abierta
  • ❌ No hay integración stock-ventas — vender no descuenta inventario
  • ❌ Servicios hardcodeados — solo funciona para barbería

¿Qué falta para 30 días de uso continuo?

  • ✅ Corregir los 6 bugs críticos (innegociable)
  • ✅ Hacer configurables los servicios/productos
  • ✅ Integrar inventario con ventas (descuento de stock)
  • ✅ Agregar confirmaciones antes de eliminar
  • ✅ Firestore Security Rules básicas (scope por companyId)
  • ✅ Logout visible en el layout
  • ✅ Estabilidad básica — que no pierda datos

Top 10 tareas a hacer primero

  1. Renombrar pge.tsx → page.tsx 5 min
  2. Renombrar carpeta singup → signup 5 min
  3. Fijar cookie name (middleware o authToken) 15 min
  4. Agregar type="password" en signup 2 min
  5. Crear .env.local + mover Firebase a variables 30 min
  6. Escribir Firestore Security Rules básicas 4 h
  7. Hacer servicios configurables desde Firestore 4 h
  8. Integrar stock inventario → ventas 3 h
  9. Agregar confirmación antes de eliminar 2 h
  10. Agregar logout en sidebar del layout principal 30 min

Tareas que NO haría todavía

  • WhatsApp real (16 h+) — demasiado complejo para MVP
  • Notificaciones push — nice-to-have, no MVP
  • Paginación Firestore — no es problema hasta 500+ registros
  • Roles multi-usuario — MVP es 1 negocio con 1 dueño
  • Facturación electrónica — depende del país y regulación
  • Exportación PDF/CSV — no esencial primeros 30 días
  • Tema claro/oscuro — ya tiene dark mode
  • Tests automatizados — primero que funcione
